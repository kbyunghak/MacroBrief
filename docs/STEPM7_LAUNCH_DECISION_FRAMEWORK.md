# Step M7 Launch Decision Framework v1

## Goal
Define objective go/no-go criteria for public mobile release.

## Go Criteria
- Crash-free sessions >= 99%
- D7 retention >= 20%
- Relevance positive ratio >= 70%
- No unresolved critical policy/compliance issues
- Alert freshness SLA consistently met

## No-Go / Hold Criteria
- Crash rate above threshold
- Relevance quality below 60%
- Repeated policy-risk language incidents
- Unstable ingestion causing stale alerts

## Decision Inputs
- KPI dashboard snapshot
- Top user complaints and trend
- Critical bug list
- Compliance checklist status

## Outcome Paths
1. Go launch: publish app store builds
2. Soft launch: limited geography/user cohort
3. Hold and iterate: 2-4 week remediation sprint

## Exit Criteria
- Decision criteria approved
- Launch meeting template prepared
- Ownership assigned for final decision sign-off
